
# Design Patterns and Principles

## Overview

This folder contains the implementation of various Design Patterns completed as part of the Cognizant Digital Nurture 5.0 Program.

Design Patterns are reusable solutions to commonly occurring software design problems. They help in building scalable, maintainable, and loosely coupled applications.

---

## Implemented Design Patterns

### 1. Singleton Pattern
Ensures that only one instance of a class exists and provides a global access point to that instance.

**Use Case:** Logging Utility

---

### 2. Factory Method Pattern
Provides an interface for creating objects while allowing subclasses to decide which class to instantiate.

**Use Case:** Document Management System

---

### 3. Builder Pattern
Separates the construction of a complex object from its representation, allowing step-by-step object creation.

**Use Case:** Computer Configuration System

---

### 4. Adapter Pattern
Allows incompatible interfaces to work together by acting as a bridge between them.

**Use Case:** Payment Gateway Integration

---

### 5. Decorator Pattern
Adds additional responsibilities to objects dynamically without modifying their structure.

**Use Case:** Notification System (Email, SMS, Slack)
